# gui Developer Guide

## Purpose
Implements the interactive Qt UI for composing and running LLM workflows.

## Contents
- `main_window.py`: Main shell (File menu, toolbar, status bar). Sets `WorkflowCanvas` as the central widget and parents `PropertiesPanel` directly onto the canvas as a floating overlay, pinned to the top-right corner via `setGeometry` (driven by a canvas resize event filter and the panel animation's `valueChanged` signal). This overlay approach prevents the canvas viewport from shrinking when the panel opens. Drives the panel by listening to `canvas.selection_changed` and wiring panel signals to canvas undo handlers. Run and save actions call `PropertiesPanel.commit_pending_edits()` before delegating to canvas execution/serialization paths. Load/Clear hide the panel before destructive canvas mutations so stale panel commits cannot target replaced/deleted nodes. Load catches parse/schema errors and shows a user-facing error dialog instead of crashing.
- `canvas.py`: `QGraphicsView` orchestration, pan/zoom controls, connection creation, trigger-driven run execution with permanent Start node, pre-run validation, draw-time graph warnings, workflow save/load serialization, in-memory node clipboard (Ctrl+C / Ctrl+V), and undo/redo stack (Ctrl+Z / Ctrl+Y). Exposes `on_output_line` and `on_output_cleared` callback attributes set by `MainWindow` to forward node output into the panel without coupling canvas to UI components. Also exposes `notify_node_changed(node_id)` so undo/redo title or model changes cause the panel to refresh. `load_workflow_data()` delegates parsing/validation to `workflow_io.parse_workflow_data` (no scene mutation), then clears/applies only on success. Malformed node records abort the entire load; malformed connection records are silently dropped. Provider registry lookup is via `workflow_io.get_provider_for_model`.
- `llm_node.py`: `WorkflowNode` — abstract base class shared by `LLMNode` and `FileOpNode`; owns connection bookkeeping, port proximity tests, `itemChange`, `boundingRect`, and the shared port-direction-arrow paint helper. `LLMNode(WorkflowNode)` — compact painted node (fixed height 64 px) with a dark-blue "LLM CALL" header strip and a title label in the body area. All editing is done in `PropertiesPanel`, not inline. Stores `_title`, `_model_id`, and `_prompt_text` as plain Python attributes. `StartNode` — permanent pure-trigger root node with output port only. `_GlowAnimator` singleton drives the animated "running" border effect (~33 fps QTimer, shared across running `LLMNode` instances).
- `llm_widget.py`: `ModelSelector` (overlay dropdown widget), `_ModelListWidget`, `NODE_WIDTH`, `ICON_SIZE`, provider icon helpers (`provider_icon`, `_fallback_provider_icon`, `_normalized_logo_pixmap`, `provider_company`, `get_registered_providers`), and `populate_model_selector` — a helper that fills any `ModelSelector` with all registered provider models. Used by `PropertiesPanel`.
- `file_op_node.py`: `FileOpNode` base class, `CreateFileNode`, `TruncateFileNode`, `DeleteFileNode`. Compact painted nodes (fixed height 64 px) with an accent-colored header and a title label in the body. Stores `_title` and `_filename` as plain attributes. `FileOpWidget` has been removed; editing is done in `PropertiesPanel`.
- `properties_panel.py`: `PropertiesPanel(QWidget)` — animated slide-in drawer that opens when a node is selected and closes when nothing is selected. Contains `_LLMForm` (name, model selector, prompt, output) and `_FileOpForm` (name, filename, output) inside scroll areas on a `QStackedWidget`. Animation is driven by `QPropertyAnimation` on `maximumWidth` (200 ms, OutCubic). Emits `title_committed`, `model_changed`, `prompt_committed`, and `filename_committed` signals. Exposes `maybe_append_output` / `maybe_clear_output` for streaming output from the canvas and `refresh_if_current(node)` to force a field reload when needed.
- `workflow_io.py`: Pure serialization/validation helpers: `parse_workflow_data` (validates and normalises a raw JSON dict — node errors abort, bad connections are dropped), `build_workflow_data` (serialises canvas state to a dict), and `get_provider_for_model` (registry lookup by model ID). No Qt widget state; imported by `canvas.py`.
- `connection_item.py`: Obstacle-routed directed edge between workflow node ports.
- `undo_commands.py`: `AddNodeCommand`, `RemoveNodeCommand`, `MoveNodeCommand`, `AddConnectionCommand`, `RemoveConnectionCommand`, `TitleChangeCommand`, `ModelChangeCommand`, `PasteCommand` — covers every graph mutation.
- `project_chooser.py`: Startup dialog for selecting the project folder; persists recently used folders in `.recent_folders.json` at the repo root.
- `assets/`: Static logo files used by the model selector.
- `__init__.py`: Package marker.

## Key Interactions
- `WorkflowCanvas` owns node/edge lifecycle and triggers trigger-driven execution: for **Run All**, the permanent `StartNode` is the single root — its direct children fire first. LLM nodes execute via `LLMWorker` (exec_id-tagged); file-op nodes execute synchronously. On successful completion, downstream triggers are queued via the Qt event loop (`QTimer.singleShot(0, ...)`) to avoid deep synchronous recursion in cyclic file-op-only graphs, and each deferred callback re-checks that the source→target edge still exists before triggering. Run drain completes only after both active invocations and queued child-trigger callbacks are exhausted. Cycles still run until `stop_all()` is called. Run Selected and Run From Here bypass the Start node and use the selection as the entry point instead.
- Before any run, all reachable nodes are validated with node-type rules: LLMNode requires a non-empty prompt and a model ID that resolves to a registered provider model; FileOpNode requires non-empty filename.
- `MainWindow` wires `canvas.selection_changed` → `_on_selection_changed` which calls `_panel.show_for_node(node)` for a single selected non-Start node, or `_panel.hide_panel()` otherwise. Panel signals are wired to canvas undo handlers and direct node attribute setters.
- `LLMNode` and `FileOpNode` store all user-editable metadata (title, prompt, model, filename) as plain Python attributes. The canvas reads them during execution and serialization.
- `MainWindow` accepts an optional `project_folder` string at construction, passes it to `WorkflowCanvas.set_working_directory()`, and delegates all graph behavior to the canvas. The **File** menu hosts Save Workflow, Load Workflow, and Open Project Folder actions.
- `ProjectChooserDialog` is shown at startup before the main window opens; the chosen folder is forwarded to `MainWindow`. The user can cancel the chooser and still open the window without a project folder, then pick one later via File → Open Project Folder.

## Behavior Notes
- The Start node is permanent: it cannot be deleted with the Delete key and is recreated after Clear Canvas. Its position is saved and restored with the workflow JSON.
- Cyclic graphs execute continuously: each node triggers its children on successful completion, so A→B→A loops until the user stops the workflow. Drawing a connection that would create a cycle shows a warning dialog at draw time, but the user may proceed and add it anyway.
- Nodes not reachable from Start are never executed by Run All.
- Children fire only on successful completion; an error node is a dead end and does not trigger its descendants.
- Drawing a second outgoing connection from a node that already has one shows an informational dialog explaining that the node will fan out to multiple parallel children; the user may proceed.
- Three run modes are available: **Run All** executes all nodes reachable from the permanent Start node (requires at least one connection from Start); **Run Selected** fires only the currently selected node(s) simultaneously without triggering their children; **Run From Here** fires the single selected node and all its descendants reachable from it. The Start-root restriction (reachability check and upfront validation) applies only to Run All; the other two modes validate only the nodes they will execute.
- Mouse wheel zoom is active on the canvas (zoom under cursor) with zoom clamped to prevent extreme scales, except while a model dropdown is open where wheel input scrolls the dropdown list instead of zooming.
- Canvas panning is driven by right-button drag (middle-button drag remains accepted).
- Left-drag on empty canvas uses a rubber-band rectangle to select multiple nodes at once.
- Background grid dots are rendered with a cosmetic pen so they stay visible and keep a fixed on-screen size across zoom levels.
- Scene bounds auto-expand around moved/loaded nodes with padding so corner nodes remain reachable after zooming and panning.
- Connection paths are routed around node bounding boxes (LLMNode, StartNode, FileOpNode) using obstacle-aware grid routing; route recompute runs immediately after connection creation and whenever nodes move/resize so lines avoid passing under nodes.
- Arrowheads terminate just outside the target input port, keeping direction visible instead of being hidden by the port dot.
- Node ports are direction-cued and color-differentiated: a left-side ingress arrow and right-side egress arrow protrude from the node edge to indicate flow, and each arrow hides once that side has at least one connection.
- Connection items use an expanded hit-test shape (line + arrowhead) so selecting arrows for delete is easier without rendering thicker lines.
- Selected nodes render an additional light-blue neon frame so active selection is visually obvious.
- `LLMNode` running state (`status == "running"`) displays an animated border driven by `_GlowAnimator` (pulsing halo + rotating sweep). `FileOpNode` uses its own paint-path glow math and does not register with `_GlowAnimator`.
- Deleting a node that has in-flight workers cancels those workers and moves their exec_ids to `_retired_exec_ids`. The `_active_workers` ref is kept alive until the callback fires (preventing a "QThread destroyed while still running" crash). When the callback arrives, `_on_invocation_done` detects the retirement flag, skips node mutation, and still calls `_check_drain()` so the run state transitions correctly if no other workers remain.
- Connections only define execution order; node outputs are displayed in the properties panel when the node is selected, not injected into downstream prompts.
- **Copy/paste**: Ctrl+C snapshots all selected workflow nodes (`LLMNode` + `FileOpNode`, Start excluded) and any connections whose both endpoints are in the selection into an in-memory clipboard, and resets a `_paste_count` counter to 0. Ctrl+V pushes a `PasteCommand` onto the undo stack; the command creates each node with a fresh `uuid4` node ID and a pre-assigned `_node_counter` label index; position is the original coordinates plus `40 * _paste_count` on both axes (incremented each paste, so successive pastes of the same clipboard land at distinct cascading offsets). Internal connections are recreated between the new nodes. Pasted nodes are selected and originals are deselected. Both shortcuts are suppressed when any text widget (`QLineEdit`, `QTextEdit`, `QPlainTextEdit`) has focus so normal text editing is unaffected. The clipboard persists only for the session and is not saved with workflow JSON.
- **Undo/redo**: Every graph mutation (add node, delete node, add connection, delete connection, drag move, rename title, change model, paste) is wrapped in a `QUndoCommand` and pushed onto `WorkflowCanvas._undo_stack` (limit 100). Ctrl+Z calls `undo()` and Ctrl+Y calls `redo()` — both shortcuts are suppressed when any text widget has focus. Toolbar Undo/Redo buttons show the current command name and are enabled/disabled automatically by Qt. Multi-item deletes and multi-node drags are wrapped in a macro so they undo as one step. `_undo_stack.clear()` is called on `clear_canvas()` and `load_workflow_data()` so stale history cannot refer to deleted nodes. After a title or model undo/redo, `canvas.notify_node_changed(node_id)` re-emits `selection_changed` if the node is selected, causing `MainWindow._on_selection_changed` to call `_panel.show_for_node` with fresh data.
- **File operation nodes** (`CreateFileNode`, `TruncateFileNode`, `DeleteFileNode`) are subclasses of `FileOpNode` defined in `file_op_node.py`. When executed, `canvas._fire_file_op` runs the operation synchronously on the main thread (no worker thread), writes success/error text into `node.output_text`, and then completes via `_on_invocation_done`. Validation requires a non-empty `filename` instead of a non-empty prompt and model. Serialization includes a `node_type` field (`"create_file"` / `"truncate_file"` / `"delete_file"`) so `load_workflow_data` can reconstruct the correct class via `NODE_TYPE_MAP`. Paths are resolved relative to the active project folder and rejected if absolute or outside that folder after canonicalization. `create_file` reports `Already exists` when the target file already exists and errors if the existing target is a directory/non-file path.
- **Properties panel**: `PropertiesPanel` slides in from the right when a single non-Start node is selected. The drawer uses `QPropertyAnimation` on `maximumWidth` (0 → 360 px, 200 ms); `MainWindow` connects `_anim.valueChanged` to `_reposition_panel` so each animation frame calls `setGeometry` on the panel, translating the animated width into a real geometry change since the panel is an absolutely positioned overlay child of the canvas. Editing the name field and pressing Enter/Tab emits `title_committed` which pushes a `TitleChangeCommand`. Changing the model emits `model_changed` which pushes a `ModelChangeCommand`. Prompt changes commit on prompt focus loss; filename changes commit on filename focus loss or Enter in the filename field; both write directly to node attributes (no undo entry). `commit_pending_edits()` commits visible-form edits without collapsing the panel (flushes prompt/filename first, then commits title) and is used before run/save and when switching the panel to a different node, so pending edits are applied before execution, serialization, or panel reload. Output from running nodes streams into the output area of the panel in real time via `maybe_append_output`. The panel is cleared and collapsed when canvas is cleared or workflow is loaded.
- Model selection uses a compact selector row that opens an overlay dropdown anchored to the active window (or the canvas viewport when proxy-embedded), clamps geometry to visible bounds (opening upward when needed), shows provider logos from `assets/` (Anthropic/OpenAI/Gemini), normalizes them to a 16x16 icon canvas, supports an explicit "no selection" state (`current_model_id == None`), and lazy-loads provider modules if needed. New LLM nodes default to `gemini-3-pro-preview` when available in the provider registry; otherwise they use the first registered model.
