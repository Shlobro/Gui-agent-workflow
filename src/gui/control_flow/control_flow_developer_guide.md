# control_flow Developer Guide

## Purpose
`control_flow/` holds node types whose main job is workflow coordination rather than LLM, file, or git work.

## Contents
- `join_node.py`: `JoinNode`, a compact barrier node that waits for `wait_for_count` arrivals before it releases one downstream continuation.
- `__init__.py`: Package marker.

## Execution Notes
- Join arrivals are grouped by the current split token generated by the execution engine. A single branch passing straight through does not create a new group; a fan-out creates a fresh shared join token for the sibling branches from that split.
- The execution engine tracks unmatched join arrivals separately from active workers so a workflow does not falsely report `Done.` while branches are still parked at a join.
- Each full batch of `wait_for_count` arrivals releases exactly one downstream trigger. Extra arrivals remain queued for the next batch.

## Serialization
- Join nodes persist `wait_for_count` in workflow JSON.
- `wait_for_count` is clamped to `1..9999` on load and paste.
